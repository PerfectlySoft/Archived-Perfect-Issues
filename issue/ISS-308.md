<p><p>Hi JGOH,</p>

<p>Thank you for providing this important information. Unfortunately, this issue has been confirmed as the swift bug itself. Perfect community contributes a few unofficial solutions before the swift.org solves it.</p>

<p>1. If the Xcode command tools are useless to you, please try uninstalling them.</p>

<p>2. If the command tools are still useful to some of your projects, then please try an alternative approach, i.e., block the pkgconfig from the default path by <tt>export PKG_CONFIG_PATH=</tt> to somewhere else, as below:</p>
<blockquote><p>Here's a messy workaround for getting Perfect-CURL to compile with both pkg-config and the developer tools installed. This prevents pkg-config from adding /usr/include to the header search path, allowing the compiler to use the headers in the SDK and avoiding <a href="https://bugs.swift.org/browse/SR-1567" class="external-link" rel="nofollow">https://bugs.swift.org/browse/SR-1567</a>. </p></blockquote>
<blockquote><p>(1) copy `/usr/local/Homebrew/Library/Homebrew/os/mac/pkgconfig/10.12/libcurl.pc` to the project directory</p></blockquote>
<blockquote><p>(2) change its "prefix" value to: `/Applications/Xcode.app/Contents/Developer/Platforms/MacOSX.platform/Developer/SDKs/MacOSX.sdk/usr`</p></blockquote>
<blockquote><p>(3). build the project with `PKG_CONFIG_PATH=$PWD swift build`</p></blockquote>
<blockquote><p>(4). generate the Xcode project with `PKG_CONFIG_PATH=$PWD swift package generate-xcodeproj`</p></blockquote>

<p>If you don't mind, I would like to close this issue because there is a duplicated but more concise issue raised as <a href="http://jira.perfect.org:8080/browse/ISS-316" title="Perfect-CURL crashes compiler if pkg-config is installed" class="issue-link" data-issue-key="ISS-316"><del>ISS-316</del></a>. You can follow this new one until fully resolved.</p>
